# Fitness Ledger 功能审阅说明

> 用途：提供给 ChatGPT 或产品顾问进行功能审阅。请重点评价实际使用流程、功能完整性、纠错能力和长期使用价值，不必阅读源码，也不要默认建议商业化、账号、云同步或 AI 教练。

## 1. 产品定位

Fitness Ledger 是一款 Windows 本地优先的个人健身记录工具，服务单个长期使用者。

它解决的核心问题是：用户每天只输入一段自由格式文字，程序将其整理为身体、饮食、训练、有氧和动作成长记录，同时永久保留原始输入，允许后续纠错和维护。

产品强调可靠、低摩擦、可回退和长期可读，不追求社交、激励打卡、训练计划或商业健身 App 的复杂功能。

## 2. 使用端组成

### Web 桌面端

当前主要使用界面，提供每日录入、Review、记录查看与编辑、动作成长、动作词典和数据检查。

### Python 本地桌面端

保留完整维护能力和兼容入口，可管理相同数据库。Web 与本地端共享解析、保存和动作词典写入逻辑。

### Mobile Viewer

手机浏览器只读查看端，用于健身房查看当天记录、搜索动作及查看动作历史。目前不承担正式编辑。

## 3. 每日录入流程

1. 用户在 Daily Entry 输入一天的自由文字。
2. 文本可包含日期、体重、排便、热量、蛋白质、碳水、脂肪、训练部位、多个动作、每组重量/次数/组数、有氧、饮食和备注。
3. Parse & Review 将文本解析为结构化记录。
4. Review 首先显示动作识别与映射，因为这是每次录入最需要人工确认的部分。
5. 用户可以确认已有动作、把新称谓映射到已有动作、加入新动作或只保留原始记录。
6. Review 后续展示核心摘要及 Body、Diet、Training 字段，允许保存前修正。
7. Confirm & Save 创建备份后写入正式数据库。
8. 原始每日文字始终保留，不会因结构化字段修改而丢失。

## 4. 重复日期处理

重复日期不会静默保存，必须选择：

- 覆盖：替换当天 Body、Diet、Training 和对应动作历史；旧原始输入保留并标记为已被替代。
- 追加训练：只新增同日第二次训练和动作历史，不重复写入 Body、Diet。
- 取消：不写入任何内容。

## 5. Body Records

记录并展示：

- 日期
- 体重
- 排便
- 当天训练部位
- 有氧
- 身体/当日备注

支持搜索、筛选、查看详情和直接编辑。修改不会删除记录中未展示的兼容字段。

## 6. Diet Records

记录并展示：

- 日期
- 总热量
- 蛋白质、碳水、脂肪
- 饮食内容摘要
- 饮食备注

列表强调宏量营养素摘要，完整饮食内容在详情中查看。已有记录可以在 Web 中修改。

## 7. Training Records

记录并展示：

- 日期及训练序号
- 训练分部
- 标准化动作摘要
- 训练备注
- 当日原始训练段落

动作备注和全局训练备注分开保存。已有 Training 记录可以在 Web 中修改。

## 8. Movement Progress

动作成长以永久 movement_id 为基础，不依赖可能变化的显示名称。

主要功能：

- 按训练部位浏览动作
- 按使用频率优先展示
- 搜索中文名、英文名和别名
- 查看动作最近训练变化
- 查看负重/次数/训练容量趋势
- 查看历次重量、次数、组数、顺序和备注
- 编辑具体动作历史记录

停用动作不会在本地或 Web 的 Movement Progress 中显示，但其历史和别名保留，后续匹配记录仍可写入；重新启用后恢复展示。

## 9. Movement Dictionary

动作词典用于统一不同称谓，避免同一动作被拆成多条成长记录。

支持：

- 新建动作
- 修改中英文标准名
- 修改训练部位、类别、器械和备注
- 添加或删除别名
- 启用或停用
- 查看结构化历史数量
- 明确确认后删除动作及其结构化历史

别名更新后会扫描 CUSTOM 动作和曾经跳过的原始记录，尝试把匹配历史归并到正式动作。删除动作不会删除每日原始输入。

## 10. Data Check

使用确定性规则检查数据质量，不做 AI 判断。

目前关注：

- 缺失关键字段
- 重复日期
- Training/Notes 字段污染
- 未识别动作
- 缺失组数
- 同日重复动作历史
- CUSTOM 动作过多

问题可以查看详情；本地端支持确认后隐藏检查项。Web 端的数据检查写入和撤销功能仍属于待迁移范围。

## 11. 已有记录纠错

Web 端现已支持：

- 编辑 Body 记录
- 编辑 Diet 记录
- 编辑 Training 记录
- 编辑动作历史中的顺序、组数、有氧参数、备注和原始动作细节
- 编辑动作词典

每日完整原始输入不提供直接覆盖式编辑，因为它承担审计和恢复来源的作用。

## 12. 数据安全与回退

- 正式数据保存在本地 JSON 文件中。
- 所有写入使用临时文件验证后原子替换。
- 保存和编辑前创建 tracker 与动作词典的成对备份。
- 桌面与 Web 共用跨进程写锁，避免同时写入。
- 写入失败时保留或恢复原文件。
- GitHub 保存程序代码、架构、规范和维护记忆，不上传个人正式数据库。
- 个人数据通过本地 `data/backups/` 回退；程序版本通过 GitHub commit 回退。

## 13. 当前明确暂缓

- 云同步、账号与多人使用
- 商业化功能
- AI 饮食评价或 AI 教练
- 自动训练计划
- 智能手表和体脂秤同步
- 大型 Dashboard
- 复杂周报/月报
- 手机端正式写入

## 14. 当前已知限制

- Mobile Viewer 依赖本地服务，电脑关闭或不在可访问网络时无法使用。
- Web Undo 尚未迁移，主要撤销入口仍在本地桌面端。
- Web Data Check 的确认隐藏/修正跳转尚未完全迁移。
- 解析依赖相对规范的文本结构，极不规则的表达仍可能需要 Review 修正。
- JSON 适合当前单人数据量，但不是面向大规模、多用户并发设计。
