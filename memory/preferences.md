# Preferences

## Communication

- Prefer direct, structured answers in Chinese unless another language is requested.
- Start with the conclusion for long explanations, then expand into steps and tradeoffs.
- Keep implementation advice actionable and specific.

## Engineering

- Prefer small, verifiable changes over broad rewrites.
- Preserve existing project style and conventions.
- Record meaningful decisions in `decisions/` when they affect future work.

## Review Cadence

- Review durable memory weekly.
- Archive outdated nodes instead of deleting them immediately.
