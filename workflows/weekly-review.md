# Weekly Review Workflow

Use this workflow once per week to convert scattered notes into durable memory and action items.

## Inputs

- Recent chat summaries
- GitHub Issues and PRs
- Local notes
- Unfinished ideas
- Completed decisions

## Steps

1. Collect notes from the week.
2. Identify durable information:
   - repeated preferences
   - project facts likely to matter later
   - decisions with future consequences
   - reusable procedures
3. Update existing memory nodes before creating new ones.
4. Create ADR files for decisions that changed future behavior.
5. Convert actionable tasks into GitHub Issues.
6. Mark outdated memory as `status: archived` when it no longer applies.

## Output

Create or update:

- `memory/*.md`
- `decisions/*.md`
- GitHub Issues, if the repository is connected
