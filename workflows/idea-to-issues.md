# Idea To GitHub Issues Workflow

Use this workflow when a rough idea needs to become concrete implementation work.

## Steps

1. Restate the idea in one sentence.
2. Identify the target outcome.
3. Split the idea into deliverable slices.
4. For each slice, create an issue with:
   - problem
   - proposed outcome
   - acceptance criteria
   - risks or unknowns
5. Add labels such as `memory`, `automation`, `skill`, `docs`, or `experiment`.
6. Link related memory nodes and decisions.

## Issue Size Rule

An issue is too large if it cannot be reviewed or verified independently. Split it until each issue has one clear completion condition.
