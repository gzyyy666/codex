# Project-First Python Learning And Tool Building

Use this workflow when teaching, learning, or building practical Python tools in small steps.

## Goal

Turn a concrete need into a runnable Python tool, while keeping the learning path practical and easy to review later.

## Steps

1. Pick one useful target behavior instead of a generic syntax drill.
2. Show a runnable example first so the learner can see the end state.
3. Identify the variable points:
   - input source
   - data shape
   - output form
   - UI or interaction mode
4. Assign a near-analogous exercise that uses the same structure with different content.
5. Review the learner's code against the target behavior and explain the gap.
6. Extend the tool into a real project by splitting core logic from the interface when needed.
7. For desktop tools, default to a direct-launch main window and avoid extra chooser screens unless they solve a real problem.
