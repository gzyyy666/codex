# Node Taxonomy

Use this taxonomy to decide where durable context belongs.

## memory/

Use for stable personal preferences, project context, collaboration notes, reusable background, and facts that are likely to help future tasks.

Examples:

- Writing style preferences
- Active project index
- Non-sensitive collaboration context
- Tooling preferences

## decisions/

Use for decisions with future consequences. Prefer ADR format.

Examples:

- Choosing GitHub as the memory layer
- Deciding to create a custom Skill
- Standardizing how weekly reviews are run

## workflows/

Use for repeatable procedures.

Examples:

- Weekly review
- Idea to GitHub Issues
- Research note to structured summary

## templates/

Use for reusable file shapes.

Examples:

- Memory node template
- ADR template
- GitHub Issue template
